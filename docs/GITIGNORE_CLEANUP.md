# Git Ignore & Repository Cleanup

## Summary

This document explains the `.gitignore` improvements and cleanup process for the Fourcast repository.

## What Was Fixed

### 1. Enhanced `.gitignore`
The `.gitignore` file has been reorganized and enhanced with:

- **Clear sections** with headers for easy navigation
- **Comprehensive coverage** of all common ignore patterns
- **AI coding assistant directories** (.qoder, .qodo, .trae, .zencoder, .zenflow, .cursor, .windsurf)
- **Database files** (*.db, *.db-shm, *.db-wal)
- **Blockchain CLI configs** (.aptos, .movement, movement-key.txt)
- **Package manager locks** (pnpm-lock.yaml, yarn.lock, package-lock.json)
- **OS-specific files** (.DS_Store, Thumbs.db)
- **Build artifacts** (.next, dist, build)

### 2. Files Currently Tracked That Shouldn't Be

The following files are currently tracked in git but should be ignored:

#### SDK Dependencies (Large)
- `sdk/node_modules/` - ~1000+ files
  - **Why**: Dependencies should be installed via `npm install`, not committed
  - **Impact**: Bloats repository size, causes merge conflicts
  - **Fix**: Run cleanup script to remove from tracking

#### Database Files
- `fourcast.db`, `fourcast.db-shm`, `fourcast.db-wal`
  - **Why**: Database files are environment-specific and auto-generated
  - **Impact**: Contains local data, causes conflicts between developers
  - **Fix**: Run cleanup script to remove from tracking

#### OS Metadata
- `.DS_Store` files (macOS)
  - **Why**: OS-specific metadata, not part of the project
  - **Impact**: Pollutes repository with irrelevant files
  - **Fix**: Run cleanup script to remove from tracking

#### IDE Configurations
- `.qoder/`, `.qodo/`, `.trae/`, `.zencoder/`, `.zenflow/`
  - **Why**: Personal IDE settings, not project configuration
  - **Impact**: Different developers use different tools
  - **Fix**: Run cleanup script to remove from tracking

#### Build Artifacts
- `tsconfig.tsbuildinfo`, `next-env.d.ts`
  - **Why**: Auto-generated by TypeScript/Next.js
  - **Impact**: Regenerated on every build
  - **Fix**: Run cleanup script to remove from tracking

## How to Clean Up

### Option 1: Automated Cleanup (Recommended)

Run the cleanup script:

```bash
./scripts/cleanup-repo.sh
```

This will:
1. Remove all ignored files from git tracking
2. Keep files on your local filesystem
3. Show you what was removed
4. Provide next steps

### Option 2: Manual Cleanup

If you prefer to do it manually:

```bash
# Remove SDK dependencies
git rm -r --cached sdk/node_modules

# Remove database files
git rm --cached fourcast.db fourcast.db-shm fourcast.db-wal

# Remove OS files
find . -name .DS_Store -print0 | xargs -0 git rm --cached

# Remove IDE configs
git rm -r --cached .qoder .qodo .trae .zencoder .zenflow

# Remove build artifacts
git rm --cached tsconfig.tsbuildinfo next-env.d.ts

# Commit changes
git commit -m "chore: cleanup gitignore and remove tracked files"
```

## After Cleanup

### 1. Verify Changes

```bash
git status
```

You should see:
- Modified: `.gitignore`
- Deleted: All the files that were removed from tracking

### 2. Commit Changes

```bash
git commit -m "chore: improve gitignore and remove tracked artifacts"
```

### 3. Push Changes

```bash
git push origin main
```

### 4. Team Notification

After pushing, notify your team:

> "I've cleaned up the repository and improved .gitignore. After pulling, you may see deleted files in git status - this is expected. The files still exist locally, they're just no longer tracked. Run `npm install` in the sdk/ directory if needed."

## Best Practices Going Forward

### Before Committing

Always check what you're committing:

```bash
git status
git diff --cached
```

### Adding New Files

Before adding new files, check if they should be ignored:

```bash
# Check if file matches ignore patterns
git check-ignore -v <filename>
```

### Environment Variables

- Never commit `.env` or `.env.local`
- Always update `.env.local.example` when adding new variables
- Use placeholder values in the example file

### Dependencies

- Never commit `node_modules/` directories
- Always use `package.json` and lock files
- Document any special installation steps

### Database Files

- Never commit `.db` files
- Use migrations for schema changes
- Document database setup in `docs/SETUP_GUIDE.md`

### IDE Configurations

- Only commit project-wide settings (like `.vscode/settings.json` if needed)
- Never commit personal IDE preferences
- Use `.editorconfig` for consistent formatting

## Troubleshooting

### "File still showing in git status after cleanup"

The file is still on your filesystem (which is correct). Git is just showing it as "deleted" from tracking. Commit the change to finalize.

### "SDK not working after cleanup"

Run `npm install` in the `sdk/` directory:

```bash
cd sdk
npm install
cd ..
```

### "Database errors after cleanup"

The database will be auto-created on first run. If you need seed data, check `scripts/` for setup scripts.

### "Accidentally committed ignored file"

Remove it from the last commit:

```bash
git rm --cached <filename>
git commit --amend --no-edit
```
